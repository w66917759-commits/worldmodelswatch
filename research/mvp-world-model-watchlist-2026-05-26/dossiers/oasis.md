## Source-Backed Facts

- The official project page describes Oasis as an experiential, realtime, open-world AI model.
- Oasis takes user keyboard input and generates an interactive video experience frame by frame, including gameplay-like physics, rules, and graphics.
- The project page says the team released code and weights for a 500M parameter model that can run locally, while the live demo uses a larger checkpoint.
- The Hugging Face model card describes Oasis 500M as a downscaled version with inference code for action-conditional frame generation.

## MVP Angle

Use Oasis as the clearest example of a real-time playable world model with public code and weights. It is useful for comparing action-conditioned generation, latency, and "world model vs game engine" framing.

## Caveats

- The open release is the 500M model, not necessarily the same capability as the hosted live demo.
- It is still a technical demo with visible long-horizon consistency and precision-control limits.
- The demo resembles Minecraft, but should be described as an AI-generated interactive experience rather than a full production game.
