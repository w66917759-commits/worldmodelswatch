Source policy:

- Prefer official product pages, documentation, project repositories, model
  cards, papers, and company posts.
- Do not use secondary coverage to upgrade availability claims.
- Keep API access, product access, open-source release status, and research
  preview status separate.
- Check live pricing, waitlists, subscriptions, region availability, and API
  access again before publishing fresh claims.
